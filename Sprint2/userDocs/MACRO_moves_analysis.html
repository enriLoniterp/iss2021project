MACRO moves analiys

MACRO moves are used to aggregate transport trolley steps into a single task easy to understand. These MACRO moves are defined as follow:
  • moveToIn: sequence of moves from transport trolley's current position to INDOOR area;
  • moveToOut: sequence of moves from transport trolley's current position to OUTDOOR area;
  • moveToSlot: sequence of moves from transport trolley's current position to a specified point of the map;
  • moveToHome: sequence of moves to transport trolley's current position to HOME.
These moves represent messages from the caller to the transport trolley to make the robot (virtual or real) move. The interaction for sending these messages can take place with different logics (represented as qak model to make them machine understandable):
  • request-response(req-resp.qakt): the caller make a request and expects a reply. In this case, it is suggested to use a non-blocking model to allow the caller to complete other tasks;
  • dispatch-event(disp-event.qakt): the caller send a dispatch and the trolley emits an event as soon as it successfully complete the MACRO move or when it encounter an obstacle. This solution is less recommended since the described interaction is 1 to 1;
  • dispatch-observable(disp-obs.qakt): the caller send a dispatch and observe trolley status to check the correctness of the move. This allows the caller to complete other tasks and get notified when the status of the trolley changes; In every scenario, the information relative to the outcome of the macro move contains the current position of the trolley as spatial coordinates (e.g. (1,5)). These model allows an easier future implementation of unit test.
